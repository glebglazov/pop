~ Queue
  A daemon that supervises per-**repository** Task-set draining, fanning `pop tasks implement <set>` runs out concurrently across registered repositories into tmux. The scheduling unit is **Repository identity**, not the picker **Project**: all of a repo's worktrees share one identity and one **Task storage**, so the Queue collapses them and dispatches at most one drain per idle repository per Ready set — never once per worktree. It targets one specific not-currently-running Ready set, never no-argument implement, which would re-pick a running set. Each repository drains serially by local Task set priority, enforced by the **Runtime execution lock**; repositories run in parallel. It routes to one representative checkout per repo: an explicit **Queue base** when set, otherwise the repo's git main worktree (any non-bare repo). A **bare** repo has no git main worktree, so it must name a Queue base or the Queue refuses to schedule it (it never guesses a checkout); per-set routing comes from a **Worktree binding**. Global cross-project priority ordering is a non-goal.
  was: A daemon that supervises per-project Task-set draining, fanning `pop tasks implement <set>` runs out concurrently across registered projects into tmux. It targets one specific not-currently-running Ready set per idle project, never no-argument implement, which would re-pick a running set. Each project drains serially by local Task set priority, enforced by the **Runtime execution lock**; projects run in parallel. Global cross-project priority ordering is a non-goal.

+ Repo override
  A section in the global `config.toml` — `[repo."<path>"]`, keyed by any path that canonicalizes to a **Repository identity** — carrying the per-repo behaviour subset (the `.pop.toml` **RepoConfig** schema: `worktree_ready`, `auto_merge_clean`, plus **Queue base**) at **higher priority** than the repo's `.pop.toml`. Resolution is global override → `.pop.toml` → built-in default. The asymmetry is deliberate: global config may express everything `.pop.toml` can (so a user can flip behaviour without committing to the repo), but `.pop.toml` never gains global-only settings (the project registry, queue agent rotation, machine-global daemon knobs).

  avoid: project entry override, glob-scoped behaviour

+ Queue base
  A per-checkout boolean (`queue_base = true`) set inside a **Repo override** block keyed by that one checkout's path, marking the single checkout that anchors a repository's Queue work: in-place drains run and commit there, and **Worktree set** checkouts fork from and integrate back to its HEAD. It is the deterministic representative the Queue routes to, and it applies to any layout — explicitly designating a non-bare repo's base when its **git main worktree** is not the wanted one. When unset, a non-bare repo falls back to its git main worktree (the primary working tree, derived even when it has linked worktrees); a **bare** repo has no such fallback and is refused, not guessed. It lives only in global config — keyed by a machine-specific checkout path, it cannot live in a branch-riding `.pop.toml`. Resolution order: per-set [[Worktree binding]] → `queue_base` → git main worktree (non-bare only) → refuse.
  under: Configuration
  avoid: queue_bare_base, trunk checkout, main worktree, default worktree

~ Worktree binding
  A durable association between one **Task set** and one git checkout for that set's execution, recorded in **Queue daemon state** as a map from **Repository identity** plus **Task set identifier** to the bound checkout path and branch. It is the Queue's universal per-set drain router — consulted for any repository, not only **Worktree-ready** ones. A binding carries a `Provisioned` bit recording who owns the checkout's teardown: **managed** (pop ran `git worktree add` under its data dir; pop tears the checkout and branch down on integration or **Abandon worktree**) versus **adopted** (a human pointed an existing, owned checkout at the set via **Bind worktree**; pop drains into it but never deletes it — abandon only forgets the association). Bindings default to adopted/never-delete, so a hand-written or unrecognized binding can never trigger a directory deletion; pop deletes only what it demonstrably created. A managed binding's checkout lives at a stable path derived from the set identifier and persists across drain exits, failures, and supervisor restarts so re-spawns resume the same branch rather than forking afresh. If a binding's checkout is missing or no longer registered with git, pop refuses to spawn and directs the human to repair git state or **Abandon worktree** — it never silently re-provisions. **Archive** does not release a binding.

+ Bind worktree
  The human act of pointing an existing, human-owned git checkout at a **Task set** so the Queue drains that set there — `pop queue bind-worktree <set>`, run from inside the target checkout. It creates an **adopted** [[Worktree binding]] (pop never deletes the checkout). Sibling to **Abandon worktree**; both are Queue-state mutations runnable without the daemon. Refuses to re-point a set that is already bound elsewhere without `--force`, and never re-points a set holding a live **Runtime execution lock**.
