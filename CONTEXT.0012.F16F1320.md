+ Orphaned Task set
  A Task set whose **Worktree binding** points at a checkout that no longer exists on disk — the human deleted the bound worktree. Detected by a cheap filesystem stat of the binding's **Runtime path** during the dashboard's fork-free build, never a git probe. A set with no binding can never be orphaned. Orphaned is a derived runtime fact like **Picked-up Task set**, not a persisted status and orthogonal to **Task set status** (a set of any status may be orphaned). Drain already refuses an orphaned set (the bound checkout is gone); the **Queue dashboard** surfaces the condition by clustering all orphaned sets into a dedicated sort tier and leaves cleanup to **Archive**. Distinct from **Orphaned task storage** (a whole repository's storage whose repo path is gone).
  avoid: stale binding, dangling worktree, broken checkout
  under: Tasks

+ Action menu
  The layered command overlay opened with `a` in the **Queue dashboard** and its drill-downs, the single home for every row-acting verb. Opening it leaves only navigation at the top level (list movement, back/exit, filter, and `l`/Enter to descend); all mutating and side-tripping verbs move one keystroke deeper into the menu, which lists only the actions applicable to the cursored row and is dismissed with `esc`. It renders anchored to the cursor (below it, or above when the cursor sits low). Each view exposes the verbs that make sense at its altitude: the set-level menu over a dashboard row (drain, integrate, bind, unbind, auto-drain, preview, unpark, runtime shell, **Archive**), and the task-level menu in both the **Task set detail view** and the **Task text peek** (**Complete task**, **Open task**, **Skip**, filtered to the cursored task's status). Verbs keep the letters they had as flat shortcuts, so the menu is discoverable without slowing a practised operator.
  avoid: command palette, context menu, action layer, popup
  under: Tasks

~ Queue dashboard
  The interactive `pop queue dashboard` TUI — the primary hands-on surface for starting and managing **Queue** work, sibling to the **Project picker** and **Worktree picker**. It is machine-global like `pop queue status`, scanning every registered repository's **Task storage** and rendering one row per non-archived **Task set** that still has outstanding queue-actionable state, excluding only a concluded **Done Task set**. Each row shows the derived **Task set status**, the set's branch, a live **Picked-up** drain indicator, and an **Auto-drain** badge. Rows sort into membership tiers — picked-up (running), then **Auto-drain**, then **Orphaned Task set**s, then the rest — with the rest grouped by project name and **Unverified**/Done rows sinking to the bottom of each project. Navigation stays at the top level (list movement, `gg`/`G`, `h`/left/`esc` to go back or exit, `/` to filter, and `l`/Enter to open the **Task set detail view**); every row-acting verb lives behind the **Action menu** (`a`). `q` and the former `s` shortcut are intentionally unbound.

~ Task set detail view
  The full-screen interactive drill-down entered with `l` or Enter from the **Queue dashboard**, replacing the table until dismissed with `h`/left/`esc`. It lists the focused **Task set**'s tasks, supports Vim-style list movement including top and bottom (`gg`/`G`), and opens a **Task text peek** for the cursored task with `l` or Enter. Its task-level verbs — **Complete task**, **Open task**, **Skip** — live behind the **Action menu** (`a`), filtered to the cursored task's status and applied without a separate confirmation.

~ Task text peek
  A nested view inside the **Task set detail view** that shows the full markdown text of the cursored task file from **Task storage**. It is opened with `l` or Enter from the task list, supports Vim-style scrolling (`j`/`k`, `ctrl-d`/`ctrl-u`, `gg`/`G`), and is dismissed with `h`/left/`esc`, returning to the same **Task set detail view**. It carries the same task **Action menu** (`a`) as the detail view, so **Complete task**, **Open task**, and **Skip** can be applied to the previewed task without leaving the peek.
