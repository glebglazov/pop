~ Queue dashboard
  The interactive `pop queue dashboard` TUI — the primary hands-on surface for starting and managing **Queue** work, sibling to the **Project picker** and **Worktree picker**. Machine-global like `pop queue status`, it scans every registered repository's **Task storage** and renders one row per non-archived **Task set** with outstanding queue-actionable state, plus **Done Task set**s that still hold a managed **Worktree binding** (shown as a clean-up reminder until archived or unbound). Columns: PROJECT, TASK SET, a protected **FLAGS** column (`AD` = **Auto-drain**, `OR` = orphaned checkout), STATUS, WORKTREE (bound branch, `[managed wt]`, or `needs bind`), and DRAIN. On narrow panes elastic columns (WORKTREE, DRAIN, then STATUS) shrink so the table fits the terminal width; FLAGS never drops. Keys: `i` opens the **Drain target picker** for an unbound set then drains, or resumes silently in the bound checkout for a bound one; `b` bind or create a worktree in advance (without draining); `U` unbind (forget-only); `a` toggle **Auto-drain**; `p` preview the working pane; `gg`/`G` move to top/bottom; `h`/left/`esc` back or exit; `l`/Enter open the **Task set detail view**. The former `I` integrate key is removed; `q` and the former `s` shortcut are intentionally unbound.
