+ Captured verify stream
  The timestamped raw agent stream recorded for one **Verifier** invocation and kept among **Task artifacts** under `streams/_verify/<work-sha>/`, accumulating as `attempt-NNN.jsonl.gz` files in the same gzipped JSONL envelope as a **Captured attempt stream** (header, timestamped events, footer). Each structured adapter-mode invocation in a verify run gets its own file — including quota-paused fall-through attempts and runs whose output was unparseable — not only the invocation whose text was parsed into a **Verify verdict**. The header carries `work_sha` and `phase: verify`; the invocation whose output was parsed may annotate its footer `reason` with `verdict: PASS|FIXABLE|NEEDS-HUMAN`. Persistence is best-effort and never blocks verification; the **Verify verdict** in the drain store stays the State cache and does not point at stream paths. Plain-output and custom-command invocations are not recorded. A cache hit that reuses an existing verdict at the current work SHA runs no agent and writes no new file.
  _Avoid_: Captured attempt stream, verify log, verification transcript

~ Attempt stream replay
  The `pop tasks stream TASK_SET[/FILE.md]` command — the read-only lens over **Captured attempt stream**s and **Captured verify stream**s. It supersedes and retires the earlier `pop tasks timings` lens, of which it is a strict superset. For each structured run it renders the full **Attempt timing breakdown** as a header (including read-time-derived token spend), then the event sequence as human- and agent-legible text: per event a `+Xs` offset, assistant text, tool invocation name and arguments, and tool result, with oversized tool payloads truncated to a head+tail excerpt. It is pure read-side telemetry: it captures nothing new, deriving everything from stored streams, and never mutates. Task attempts: all of a task's implement attempts print chronologically; a bare set concatenates every task in manifest order. Verify runs: `TASK_SET/_verify` prints every verification attempt across all work SHAs in chronological order; `TASK_SET/_verify/<work-sha>` narrows to one SHA's attempts. `--last` narrows to the most recent attempt, `--full` disables payload truncation, and `--raw` dumps the stored gzipped JSONL verbatim. Like `timings` it is a read-only snapshot verb that resolves archived sets by explicit identifier.
