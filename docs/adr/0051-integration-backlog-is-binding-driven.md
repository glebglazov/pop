# Integration backlog membership is binding-driven, not mergeability-record-driven

> **Tombstoned.** The Integration backlog was removed entirely by [ADR-0070](0070-worktree-set-integration-is-removed-merging-is-the-humans-own-concern.md). There is no merge reconciliation in pop, so there is no backlog to derive. A Done set holding a managed Worktree binding now surfaces only as a clean-up reminder on the Queue dashboard until it is archived or unbound.
