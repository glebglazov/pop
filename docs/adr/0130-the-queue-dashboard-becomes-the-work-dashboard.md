# The Queue dashboard becomes the Work dashboard

With Wayfinder Maps ([ADR-0129](0129-wayfinder-maps-are-a-first-class-sibling-of-task-sets.md)) pop gained a second kind of ongoing work, and the question was whether it gets its own dashboard. It doesn't: the Queue dashboard's real job was always "the hands-on surface for my task work", and Maps and Task sets are two phases of one pipeline — search, then implement — so a human asking "what am I doing or planning to do" wants one table. The dashboard moves to a new `pop work` family as the **Work dashboard**; `pop queue dashboard` survives as a hidden compatibility alias (house precedent: `pop dashboard`, `pop select`). The Queue name and scope are otherwise untouched — `pop queue run`/`status`/`log` remain the scheduling concern, and the daemon still drains only Auto-drain Ready sets; Maps are invisible to it.

Rows interleave by project rather than splitting into planning/execution sections, so a repository's Map sits beside its Task sets and the per-project story reads in one place. Task-set rows keep their behaviour and keys unchanged; queue-verbs (`a`, `b`, `U`) are inert on Map rows. Map rows carry the work action on the row itself, symmetric with sets: `i` spawns an attended wayfinder session for the next frontier ticket — a new window in the repo's tmux session named after the Map (attended work gets a window; `pop-queue` stays reserved for unattended drains), using the default Interactive agent preset — and Enter/`l` opens a Map detail view (mirror of the Task set detail view) where `i`/Enter targets one specific ticket. Map lifecycle verbs stay in the `pop wayfinder` family (`status`, `show`, `archive`/`unarchive`). `pop work` holds the dashboard plus `pop work show-path`, the single storage-root resolver: it prints the Task-storage root, and the planning skills address both kinds of work through it — `$(pop work show-path)/tasks/<set>` and `$(pop work show-path)/wayfinder/<map>` — so there is no separate per-concept path verb; `pop tasks show-path` survives as the tasks/-specific compatibility alias. Done, abandoned, and archived Maps are hidden from the dashboard.
